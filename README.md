# CYMOE

Небольшой проект на Django для ведения записей/заметок.

## Что умеет

- Регистрация и вход/выход пользователя.
- Профиль:
  - просмотр профиля,
  - смена имени пользователя,
  - смена пароля,
  - удаление аккаунта.
- Записи (модель `Post`):
  - создание записи,
  - просмотр всех записей,
  - просмотр только своих записей,
  - просмотр записи по `id`.
- Админ-панель Django.

## Основные URL

- `/` — главная
- `/register/` — регистрация
- `/login_user/` — вход
- `/logout_user/` — выход
- `/profile/` — профиль
- `/change_name/` — смена имени
- `/change_pass/` — смена пароля
- `/delete_profile/` — удаление аккаунта
- `/create_note/` — создать запись
- `/user_note/` — мои записи
- `/view_note/` — все записи
- `/post/<id>/` — детальная страница записи (для «моих»)
- `/post_all/<id>/` — детальная страница записи (для списка «все записи»)
- `/admin/` — админка

## Технологии

- Python + Django (в проекте используется Django 6.0.2)
- База данных: SQLite (`db.sqlite3`)
- Шаблоны: `templates/`
- Статика: `blog/static/`

## Быстрый старт (Linux)

1) Создать и активировать виртуальное окружение:

```bash
python -m venv .venv
source .venv/bin/activate
```

2) Установить зависимости:

```bash
pip install -r requirements.txt
```

3) Создать файл окружения:

```bash
cp .env.example .env
```

4) Применить миграции:

```bash
python manage.py migrate
```

5) (Опционально) создать администратора:

```bash
python manage.py createsuperuser
```

6) Запустить сервер разработки:

```bash
python manage.py runserver
```

Открыть в браузере: `http://127.0.0.1:8000/`

## Переменные окружения (.env)

Проект читает `.env` из корня через `python-dotenv`.

- `DJANGO_SECRET_KEY` — секретный ключ Django (в проде обязателен)
- `DJANGO_DEBUG` — режим отладки (`1`/`0`, `true`/`false`)
- `DJANGO_ALLOWED_HOSTS` — список хостов через запятую (например: `127.0.0.1,localhost`)
